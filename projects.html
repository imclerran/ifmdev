LibECS

(Jan. 2019 - Present)

LibECS is an implementation of the Entity-Component-System architectural pattern, used to manage game objects and their properties and behaviors. The ECS pattern prevents inflexible and complex inheritance heirachies, which can sometimes lead to liskov substitution violations, where subclasses do not implement all of their parent classes behavior.

This project was born out of a need for a better way to structure game code, when writing a LibGDX bullet-hell game for the second time, in an attempt to learn from mistakes of the past.
